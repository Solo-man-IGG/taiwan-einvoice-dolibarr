# TaiwanEInvoice for Dolibarr (V1.0)

這是一個專為 **Dolibarr ERP** 設計的台灣電子發票模組，旨在提供符合中華民國財政部 **MIG 4.1 規範** 的電子發票管理解決方案。

* **官方網站**：[https://www.igg.tw/](https://www.igg.tw/)
* **主要開發者**：Solo-man (Vincent Tsai)

---

## ✨ 核心特色 (Features)

* **自動配號系統**：於發票驗證 (Validate) 時，系統自動根據年度字軌優先權進行配號。
* **資料自動對接**：營業人資訊（名稱、統編）直接抓取 Dolibarr 系統設定，無需重複填寫。
* **智慧欄位驗證**：自動處理 B2B、B2C、手機載具、自然人憑證及捐贈碼。
* **全規格 XML 支援 (MIG 4.1)**：產出之 XML 格式可直接配合財政部 **Turnkey** 軟體自主傳輸。
* **專業列印樣板**：支援 57mm 熱感紙發票證明聯與 A4 銷貨退回/折讓證明單。

## 🛠 安裝與基本使用指南

### 1. 模組部署
1. 將 `taiwaneinvoice` 資料夾放置於 Dolibarr 的 `htdocs/custom/` 目錄。
2. 登入管理員帳號，至「模組/應用程式設定」找到「台灣電子發票」並開啟。
3. 系統將自動執行 `sql/install.sql` 建立必要的資料表。

### 2. 初始化設定
* **確認系統資訊**：請確保在 Dolibarr 的「設定 > 公司/組織」中已填妥正確的 **公司名稱** 與 **專業識別碼 (統編)**。
* **設定字軌**：
    * 進入模組的「字軌管理」頁面 (`track_list.php`)。
    * 新增當期可用之電子發票字軌，並設定優先權 (Sort Order)。數字越大，系統配號時越優先使用。

### 3. 日常作業流程
* **發票開立**：在建立發票時，於前端介面選取載具類型（如手機條碼、三聯式等）。
* **驗證配號**：點擊「驗證 (Validate)」後，系統會自動在後台完成配號並產生隨機碼。
* **列印與補印**：驗證後即可點擊「列印證明聯」按鈕進行熱感紙輸出。

## 💰 進階應用：透過財政部 Turnkey 自主上傳

本模組最大的優勢在於產出的 XML 檔案完全符合財政部規範，讓企業可以**跳過加值服務商**，達成 0 月費自主傳輸。

### 1. 為何使用 Turnkey？
* **省下高額費用**：免除加值服務商的月費及按張數計費的成本。
* **數據主權**：發票資料直接從企業內部傳輸至財政部。

### 2. 操作步驟
1. **申請帳號**：至財政部電子發票整合服務平台申請開通「自行傳輸」權限。
2. **安裝 Turnkey**：下載並安裝財政部提供的 Turnkey 傳輸軟體。
3. **匯出 XML**：
   * 進入本模組的 `setup.php` 頁面。
   * 點擊「綜合打包下載 XML (ZIP)」工具，選取日期區間進行匯出。
4. **放置檔案**：
   * 解壓縮下載的 ZIP 包，將內含的 XML 檔案放置於 Turnkey 指定的傳輸目錄（通常為 `In` 資料夾）。
   * Turnkey 將會自動偵測檔案、簽章並上傳至財政部。
