# 多麗巴爾 (Dolibarr) 台灣電子發票模組 V1.0

這是一個專為 **多麗巴爾 (Dolibarr) ERP** 系統開發的繁體中文插件，旨在提供符合中華民國財政部 **MIG 4.1 規範** 的電子發票管理解決方案。

## 🏢 開發維護
* **開發單位**：璦閣數位科技 (IGG Digital Technology)
* **官方網站**：[https://www.igg.tw/](https://www.igg.tw/)
* **主要開發者**：Solo-man (Vincent Tsai)

---

## ✨ 核心特色 (Features)

* **自動配號系統**：於發票驗證 (Validate) 時，系統自動根據年度字軌優先權進行配號，流程無縫整合。
* **資料自動對接**：營業人資訊（名稱、統編）直接抓取「多麗巴爾」系統內的組織設定，無需重複填寫。
* **智慧欄位驗證**：自動處理 B2B (三聯式)、B2C (二聯式)、手機載具、自然人憑證及捐贈碼。
* **全規格 XML 支援 (MIG 4.1)**：產出之 XML 格式可直接配合財政部 **Turnkey** 軟體自主傳輸。
* **專業列印樣板**：支援 57mm 熱感紙發票證明聯與 A4 銷貨退回/折讓證明單。
* **原生繁體中文**：介面邏輯已內建繁體中文，安裝即用，無需額外維護語系檔 (.lang)。

## 🛠 安裝與基本使用指南

### 1. 模組部署
1. 將 `taiwaneinvoice` 資料夾放置於多麗巴爾系統的 `htdocs/custom/` 目錄。
2. 登入管理員帳號，至「模組/應用程式設定」找到「台灣電子發票」並開啟。
3. 系統將自動執行 `sql/install.sql` 建立必要的擴充資料表。

### 2. 初始化設定
* **確認系統資訊**：請確保在「設定 > 公司/組織」中已填妥正確的 **公司名稱** 與 **專業識別碼 (統編)**。
* **設定字軌**：
    * 進入模組的「字軌管理」頁面 (`track_list.php`)。
    * 新增當期可用之電子發票字軌，並設定優先權 (Sort Order)。

### 3. 日常作業流程
* **發票開立**：在建立發票時，於前端介面選取載具類型。
* **驗證配號**：點擊「驗證 (Validate)」後，系統會自動在後台完成配號。
* **列印證明聯**：驗證完成後即可進行熱感紙輸出。

## 💰 進階應用：透過財政部 Turnkey 自主上傳 (省錢方案)

本模組最大的優勢在於產出的 XML 檔案完全符合財政部規範，讓企業可以**跳過加值服務商**，達成 0 月費自主傳輸。

### 1. 為何使用 Turnkey？
* **節省維護成本**：免除加值服務商收取的年費、月費及按張數計費的傳輸費。
* **數據主權**：發票資料直接從企業內部傳輸至財政部，安全且自主。

### 2. 操作步驟
1. **申請帳號**：至財政部電子發票整合服務平台申請開通「自行傳輸」權限。
2. **安裝軟體**：下載並安裝財政部提供的 Turnkey 傳輸軟體。
3. **匯出與傳輸**：
   * 進入模組設定頁面 (`setup.php`)，點擊「綜合打包下載 XML (ZIP)」工具。
   * 將下載的 XML 檔案放置於 Turnkey 指定的傳輸目錄（通常為 `In` 資料夾）。
   * Turnkey 將會自動執行簽章並上傳至財政部。
